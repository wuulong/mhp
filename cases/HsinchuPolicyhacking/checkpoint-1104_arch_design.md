# Hackathon Kickoff Report - 2025年11月3日 星期一

### 挑戰理解

**核心問題：**
新竹市面臨日益嚴重的詐騙問題，尤其在114年4月，詐騙案件數量和財損金額均居全國之冠。這不僅影響市民財產安全，也可能導致青年在不知情下成為詐騙共犯。此外，政府服務效率提升與資料安全隱私保護，以及市民對公共議題的參與度，也是重要的挑戰。

**受影響對象：**
*   **市民大眾：** 尤其是各年齡層的詐騙受害者，以及對公共事務參與度較低的市民。
*   **青年族群：** 容易因不熟悉風險而捲入詐騙案件，同時也缺乏參與在地發展的平台。
*   **市府單位：** 面臨提升服務效率、確保資料安全與隱私的挑戰。

**問題解決的重要性：**
*   **提升市民福祉：** 減少詐騙案件能直接保護市民財產，提升居住安全感。
*   **促進社會參與：** 透過創新方式鼓勵市民參與公共事務，共同打造宜居城市。
*   **優化政府效能：** 運用新興科技（如AI）提升市府運作效率，提供更優質的公共服務。
*   **青年發展：** 提供青年發聲與參與的平台，激發其對在地發展的熱情與貢獻。

**解決問題可能帶來的積極變化：**
*   新竹市成為更安全、更智慧、更具公民參與活力的城市。
*   市民對政府的信任度提升，政府服務更貼近民意。
*   青年人才留在新竹，為城市發展注入新動力。
*   形成政府與市民共創共榮的良好循環。

### 初步概念發想與腦力激盪

**主題一：全民反詐，安心家園**
*   **AI 詐騙辨識系統：** 開發結合自然語言處理（NLP）和機器學習的AI模型，即時分析通話、簡訊、社群媒體內容，辨識詐騙模式並發出警示。
*   **區塊鏈防偽平台：** 針對投資詐騙，建立基於區塊鏈的投資項目驗證平台，確保資訊透明和不可篡改。
*   **互動式反詐宣導遊戲：** 開發手機遊戲或網頁遊戲，以生動有趣的方式教育市民辨識詐騙手法。

# Problem Solving Session: AI 詐騙辨識與預警系統

**Date:** 2025年11月3日 星期一
**Problem Solver:** BMad
**Problem Category:** 公共安全 / 智慧城市

---

## 🎯 PROBLEM DEFINITION

### Initial Problem Statement

新竹市詐騙案件頻傳，市民財產損失嚴重，且現有防詐機制多為事後補救，缺乏即時預警能力，導致市民防不勝防。

### Refined Problem Statement

新竹市市民正遭受日益猖獗的詐騙威脅，尤其在電話、簡訊及網路投資等管道，導致巨額財產損失與社會信任度下降。現有防詐機制未能提供即時有效的預警，使得市民難以辨識詐騙手法。我們需要一個能夠即時辨識潛在詐騙通訊並主動預警的 AI 系統，以顯著降低詐騙發生率，保護市民財產安全，並提升整體社會的防詐意識。

### Problem Context

根據新竹市警察局數據，新竹市詐騙案件數量與財損金額居全國之冠，顯示問題的嚴重性與急迫性。詐騙手法不斷演變，市民難以應對。

### Success Criteria

詐騙案件發生率降低 20%；市民對詐騙電話/簡訊的辨識率提升 30%；系統能提供 90% 以上的詐騙辨識準確率。



---

## 🔍 DIAGNOSIS AND ROOT CAUSE ANALYSIS

*   **問題是什麼？** 詐騙電話、簡訊、網路訊息難以辨識；市民財產損失；現有防詐宣導效果有限，缺乏即時預警。
*   **問題不是什麼？** 一般性的金融犯罪；市民對科技產品的普遍使用障礙；政府部門間的普遍性溝通不良。
*   **發生在哪裡？** 主要在通訊管道（電話、簡訊、社群媒體、網路平台）；新竹市各區域。
*   **不發生在哪裡？** 實體交易中面對面的詐騙；沒有通訊設備的環境。
*   **何時發生？** 隨時都可能發生，尤其在資訊不對稱、市民警覺性較低時。
*   **何時不發生？** 市民警覺性高，或已接收到明確防詐資訊時；通訊內容明顯無詐騙意圖時。
*   **誰受影響？** 新竹市所有市民，特別是長者、學生、對新興科技不熟悉的族群；金融機構、電信業者；政府。
*   **誰不受影響？** 對詐騙手法有高度辨識能力且警覺性高的人；沒有通訊設備或不使用網路的人。

**模式分析 (Patterns that emerge):**
*   問題主要集中在**通訊內容的辨識**上，這是一個資訊處理和判斷的問題。
*   詐騙的**時間和地點廣泛**，需要一個全天候、廣覆蓋的解決方案。
*   **受影響人群廣泛**，但特定群體更為脆弱，需要通用且易於理解的預警方式。
*   解決方案應聚焦於**通訊內容的即時分析與預警**，而非傳統的實體防範。

### Root Cause Analysis

*   **應用「五個為什麼」進行根本原因分析 (針對「新竹市詐騙案件頻傳，市民財產損失嚴重」)：**
    1.  **為什麼新竹市詐騙案件頻傳，市民財產損失嚴重？** 因為市民難以辨識詐騙手法，且缺乏即時有效的預警機制。
    2.  **為什麼市民難以辨識詐騙手法，且缺乏即時有效的預警機制？** 因為詐騙手法不斷演變，傳統防詐宣導更新速度慢，且缺乏科技工具進行即時通訊內容分析。
    3.  **為什麼詐騙手法不斷演變，傳統防詐宣導更新速度慢，且缺乏科技工具進行即時通訊內容分析？** 因為詐騙集團利用新興科技（如 AI、大數據）快速迭代詐騙劇本，而政府與防詐機構在技術投入和反應速度上相對滯後。
    4.  **為什麼政府與防詐機構在技術投入和反應速度上相對滯後？** 因為缺乏足夠的技術人才、資金投入，以及跨部門、跨領域的技術合作平台，導致難以快速開發和部署先進的防詐技術。
    5.  **為什麼缺乏足夠的技術人才、資金投入，以及跨部門、跨領域的技術合作平台？** **根本原因：** 缺乏一個整合性的、以科技為導向的防詐策略與生態系統，未能將學術界、產業、政府和市民的力量有效結合，共同應對不斷升級的詐騙挑戰。

### Contributing Factors

*   詐騙集團技術升級，利用 AI 等工具。
*   市民防詐知識更新不及詐騙手法變化。
*   現有防詐宣導多為被動式，缺乏主動預警。
*   政府部門間資訊共享與技術合作不足。
*   缺乏專門針對通訊內容的即時分析工具。

### System Dynamics

*   **正向回饋迴路：** 詐騙成功 -> 詐騙集團獲利 -> 投入更多資源開發新手法 -> 更多詐騙成功 (惡性循環)。
*   **負向回饋迴路 (期望)：** 成功預警 -> 市民警覺性提高 -> 詐騙成功率下降 -> 詐騙集團獲利減少 -> 減少詐騙活動 (良性循環)。
*   **延遲效應：** 詐騙手法出現 -> 造成損失 -> 宣導更新 -> 宣導見效 (時間差)。
*   **槓桿點：** 提升即時通訊內容分析能力，並建立跨部門、跨領域的技術合作平台，是打破惡性循環的關鍵槓桿點。

**1. 使用者中心設計 (User-Centered Design)**
*   **原則：** 始終將使用者的需求、行為和情境放在設計的核心。

    *   **了解目標使用者：** 特別關注長者和學生等不同年齡層的使用習慣、數位素養和對詐騙的認知程度。設計時應考慮他們可能遇到的困難。
    *   **情境化設計：** 模擬使用者在接到可疑訊息時的焦慮、急迫感，設計能快速提供幫助的介面。
    *   **易學性與易用性：** 介面操作流程應直觀，即使是初次使用的使用者也能快速上手。

**2. 清晰與簡潔 (Clarity & Simplicity)**
*   **原則：** 介面應避免不必要的複雜性，資訊傳達應直接明瞭。
*   **實踐建議：**
    *   **簡化輸入：** 文字輸入區應是介面最主要且顯眼的元素，避免過多干擾。

**3. 一致性 (Consistency)**
*   **原則：** 介面元素、互動模式和視覺風格應在整個產品中保持一致。
*   **實踐建議：**
    *   **視覺一致：** 統一使用字體、顏色、圖示風格和按鈕樣式。
    *   **互動模式一致：** 相似的功能應有相似的互動方式，例如所有確認按鈕的樣式和位置。
    *   **資訊呈現一致：** 詐騙風險等級的顏色、圖示和文字說明應在所有情境下保持一致。

**4. 回饋與引導 (Feedback & Guidance)**
*   **原則：** 系統應即時回應使用者的操作，並在需要時提供必要的引導。
*   **實踐建議：**
    *   **即時回饋：** 使用者輸入內容後，應立即顯示處理中的狀態，並在短時間內呈現辨識結果。
    *   **錯誤處理：** 如果輸入內容無效或發生錯誤，應提供清晰的錯誤訊息和解決方案。

    *   **回報確認：** 使用者提交回報後，應給予成功的確認訊息。

**5. 無障礙設計 (Accessibility)**
*   **原則：** 確保產品對所有使用者，包括身心障礙者，都能夠使用和理解。
*   **實踐建議：**
    *   **高對比度：** 文字與背景之間應有足夠的對比度，特別是對於長者或視力不佳的使用者。

    *   **語音輔助：** 考慮為關鍵資訊提供語音朗讀功能。

    *   **清晰的焦點指示：** 當使用者透過鍵盤導航時，應有明顯的焦點指示。

**6. 視覺設計 (Visual Design)**

**7. 安全性與信任 (Security & Trust)**
