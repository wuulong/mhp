**專案名稱：** AI 詐騙辨識與預警系統

**核心目標：** 降低新竹市詐騙案件發生率，提升市民對詐騙的警覺性與辨識能力，提供即時有效的預警。

**主要使用者：** 新竹市市民（尤其是容易受騙的長者、學生），警政單位。

**使用者情境與痛點：**
*   市民接到陌生電話或簡訊，內容涉及投資、中獎或假冒公務機關。
*   市民難以辨別真偽，容易因恐懼、貪婪或資訊不對稱而受騙。
*   現有防詐宣導被動，缺乏即時預警機制。

**MVP (最小可行產品) 核心功能：**
1.  **數據接收模組：** 接收文字形式的通訊內容（例如簡訊或電話語音轉文字）。
2.  **AI 詐騙辨識模組：** 運用已訓練的 NLP 模型，對通訊內容進行詐騙風險評分。
3.  **多層次預警機制：** 基於風險評分，向使用者前端（Web 介面）發出高、中、低風險警示。
4.  **簡單使用者 Web 介面：** 提供文字輸入區，顯示即時辨識結果和預警訊息。

**技術棧 (前端)：**
*   Web 技術 (HTML, CSS, JavaScript)
*   框架：React 或 Vue.js (用於構建互動式使用者介面)

**設計挑戰與優化方向：**
*   如何清晰、直觀地展示詐騙風險和預警訊息。
*   如何在有限時間內開發具備基本功能的介面。
*   清晰化預警訊息。
*   增加互動性，允許使用者對預警結果進行反饋。

**視覺與演示考量 (來自簡報準備報告，可作為設計參考)：**
*   **配色方案：** 選擇專業、清晰且具警示性的配色。例如，藍色系代表科技與信任，搭配橙色或紅色系作為警示色。
*   **圖示：** 簡潔、現代的圖示，用於代表核心功能（例如：電話、簡訊、盾牌、警示符號）。
*   **架構圖：** 簡化版系統架構圖，用圖示表示數據流、AI 模組、預警機制和使用者介面。

**綜合以上資訊，我建議的介面類型是：** 網頁介面 (Web Application)。

**設計目標：**
*   提供一個直觀、易用的平台，讓市民能快速輸入通訊內容並獲得詐騙風險評估。
*   以清晰、明確的方式呈現預警訊息，幫助使用者理解風險並採取行動。
*   提升使用者對詐騙的警覺性與辨識能力。

**核心功能 (介面呈現)：**
*   一個顯眼的文字輸入框，供使用者貼上或輸入可疑的簡訊、電話內容等。
*   一個清晰的區域，用於顯示 AI 辨識後的詐騙風險等級（例如：安全、中度風險、高度風險）。
*   針對不同風險等級，提供相應的建議或行動指引（例如：安全則無須理會；中度風險則建議查證；高度風險則建議立即報警）。
*   提供一個簡單的機制，讓使用者可以回報該預警是否準確，以協助模型優化。

**特定要求：**
*   介面應簡潔、直觀，特別考量到長者和學生等不同年齡層的使用者。
*   配色應專業且具警示性，避免過於花俏。
*   考慮使用易於理解的圖示來輔助說明功能和預警。

### 原型建立需求建議

**1. 原型功能範圍：**
*   **通訊內容輸入：** 提供一個清晰的文字輸入區。
*   **AI 詐騙風險辨識與顯示：** 能夠即時顯示輸入內容的詐騙風險等級（高、中、低）。
*   **預警訊息與建議：** 針對不同風險等級，提供簡潔明瞭的預警訊息和建議行動。
*   **基本使用者回報機制：** 允許使用者對辨識結果進行簡單的「是/否」或「準確/不準確」回報，以示範未來模型優化的可能性。

**2. 互動細節：**
*   **使用者輸入：**
    *   提供一個大型、易於辨識的文字輸入框，並有明確的提示文字（例如：「請貼上或輸入可疑簡訊/電話內容」）。
    *   支援直接輸入和貼上文字。
    *   考慮加入一個「清除」按鈕，方便使用者重新輸入。
*   **風險等級顯示：**
    *   使用**顏色區分**不同風險等級：例如，綠色代表安全，黃色代表中度風險，紅色代表高度風險。
    *   搭配**簡潔的文字說明**（例如：「安全」、「中度風險，請留意」、「高度風險，請立即警覺」）。
    *   可考慮使用**直觀的圖示**（例如：盾牌、警示符號）來強化視覺效果。
    *   結果應在使用者輸入或點擊「分析」按鈕後**即時顯示**，避免等待。
*   **預警訊息互動：**
    *   預警訊息應以**卡片或區塊**的形式呈現，內容清晰，字體大小適中。
    *   針對高度風險，可考慮加入一個顯眼的「立即報警」或「聯繫反詐騙專線」按鈕。
*   **使用者回報：**
    *   在辨識結果下方，提供簡單的選項，例如：「此結果準確嗎？」並提供「是 / 否」或「準確 / 誤判」的按鈕。
    *   點擊後可顯示一個簡短的感謝訊息，並說明回報將有助於系統改進。

**3. 預期呈現方式：**
*   **可點擊的線框圖 (Clickable Wireframe)** 或**低擬真互動原型 (Low-Fidelity Interactive Prototype)**。
*   **具體要求：** 能夠模擬從輸入內容、點擊分析、到顯示結果和回報的完整流程。不需要精美的視覺設計，但需要確保介面元素清晰可見，互動邏輯正確。
