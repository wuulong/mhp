# 解決方案開發工作流程 (Solution Development Workflow)

## 目的

本工作流程旨在指導黑客松參賽團隊進行迭代開發，包括設計、實現、測試和優化，以高效地將概念轉化為可行的解決方案。

## 如何調用

這個工作流程是 `standalone: true`，因此可以直接由使用者調用。

在 BMAD 環境中，您可以透過以下指令啟動它：

```bash
workflow /bmad/mhp/workflows/solution-development/
```

## 預期輸入

本工作流程為中互動性，將在關鍵決策點引導使用者提供輸入。使用者需要準備好：

*   已確定的解決方案願景和範圍。
*   對技術棧和開發方法的初步想法。
*   團隊成員的開發專長。

## 生成輸出

本工作流程將在執行過程中，透過對話引導團隊完成解決方案的開發。最終會生成一份摘要報告，記錄開發過程中的關鍵決策、迭代規劃和測試結果：

*   **預設輸出檔案**：`{output_folder}/solution-development-report-{{date}}.md`

## 特殊要求

*   本工作流程為中互動性，需要使用者在關鍵決策點提供確認和輸入。
*   建議在團隊已完成「黑客松啟動工作流程」後使用。
