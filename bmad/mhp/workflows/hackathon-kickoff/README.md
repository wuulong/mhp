# 黑客松啟動工作流程 (Hackathon Kickoff Workflow)

## 目的

本工作流程旨在引導黑客松參賽團隊，在專案啟動階段進行全面的規劃。它將協助團隊：

*   深入理解黑客松挑戰並定義核心問題。
*   進行初步的概念發想與腦力激盪。
*   透過設計思考聚焦於最具潛力的解決方案。
*   明確團隊成員的角色與職責分配。
*   制定初步的專案計畫與里程碑設定。

## 如何調用

這個工作流程是 `standalone: true`，因此可以直接由使用者調用。

在 BMAD 環境中，您可以透過以下指令啟動它：

```bash
workflow /bmad/mhp/workflows/hackathon-kickoff/
```

## 預期輸入

由於這是一個高互動性的意圖導向工作流程，它將主要透過與使用者的對話來獲取輸入。使用者需要準備好：

*   對黑客松挑戰的初步理解。
*   團隊成員的技能與專長資訊。
*   對專案目標和時間的初步想法。

## 生成輸出

本工作流程將在執行過程中，透過對話引導團隊完成規劃。最終會生成一份摘要報告，記錄啟動會議的關鍵決策和規劃：

*   **預設輸出檔案**：`{output_folder}/hackathon-kickoff-report-{{date}}.md`

## 特殊要求

*   本工作流程為高互動性，需要使用者積極參與對話。
*   建議在黑客松開始前，由整個團隊共同參與此工作流程。
