# 黑客松啟動工作流程驗證清單

## 總體結構

- [ ] 所有步驟都已完成，沒有遺漏。
- [ ] 清單內容清晰，易於理解。

## 1. 理解黑客松挑戰

- [ ] 團隊已明確定義核心問題。
- [ ] 已識別問題對目標受眾的影響。
- [ ] 已討論解決問題可能帶來的積極變化。
- [ ] 團隊對黑客松挑戰有共同且深入的理解。

## 2. 初步概念發想與腦力激盪

- [ ] 已生成多樣化的解決方案概念（至少 10 個）。
- [ ] 腦力激盪過程中鼓勵了自由發想，沒有過早的判斷。
- [ ] 發想出的概念已進行初步整理和分類。

## 3. 設計思考與解決方案聚焦

- [ ] 已進行使用者情境分析。
- [ ] 已識別主要使用者痛點。
- [ ] 已評估不同解決方案概念的優劣。
- [ ] 團隊已聚焦於 1-2 個最具潛力且符合黑客松目標的解決方案方向。

## 4. 團隊角色與職責分配

- [ ] 團隊成員的技能與專長已識別。
- [ ] 黑客松期間所需的關鍵角色已討論並分配。
- [ ] 每個角色都有清晰的職責範圍和預期貢獻。
- [ ] 團隊成員對自己的角色和協作方式有清晰的理解。

## 5. 專案初步規劃與里程碑設定

- [ ] 已制定初步的專案計畫：包括主要任務和預計時間分配。
- [ ] 已設定黑客松期間的關鍵里程碑。
- [ ] 已識別潛在風險並討論應對策略。
- [ ] 已建立團隊內部溝通和進度更新機制。

## 最終驗證

- [ ] 團隊對啟動流程的結果感到滿意。
- [ ] 所有關鍵決策都已記錄。
- [ ] 團隊已準備好進入黑客松的開發階段。
