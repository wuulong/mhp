# 黑客松回顧工作流程驗證清單

## 總體結構

- [ ] 所有步驟都已完成，沒有遺漏。
- [ ] 清單內容清晰，易於理解。

## 1. 設定回顧目標與安全空間

- [ ] 已明確本次回顧的目的。
- [ ] 已建立開放、誠實、無指責的討論環境。
- [ ] 團隊成員對回顧原則達成共識。

## 2. 收集數據與事件

- [ ] 已收集黑客松期間「做得好的地方」。
- [ ] 已收集黑客松期間「可以改進的地方」。
- [ ] 已收集黑客松期間「遇到的阻礙」。
- [ ] 已收集黑客松期間「學到的新事物」。
- [ ] 所有團隊成員都有機會分享。

## 3. 分析原因與洞察

- [ ] 已對「可以改進的地方」和「遇到的阻礙」進行根本原因分析。
- [ ] 已分析「做得好的地方」背後的成功因素。
- [ ] 已將分析轉化為清晰的洞察。
- [ ] 避免了表面歸因，鼓勵了批判性思考。

## 4. 制定行動計畫

- [ ] 已針對每個洞察制定具體的改進行動。
- [ ] 已確定行動的優先順序。
- [ ] 已分配行動的責任人。
- [ ] 已設定行動的預計完成時間。
- [ ] 行動計畫符合 SMART 原則。

## 5. 回顧總結與慶祝

- [ ] 所有行動計畫都已清晰記錄並達成共識。
- [ ] 每位成員都分享了最大收穫。
- [ ] 團隊成員互相表達了感謝。
- [ ] 回顧在積極和充滿希望的氛圍中結束。

## 最終驗證

- [ ] 團隊對回顧結果感到滿意。
- [ ] 所有關鍵洞察和行動計畫都已記錄。
- [ ] 團隊已準備好將學習應用於未來的合作。
