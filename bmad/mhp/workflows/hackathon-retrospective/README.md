# 黑客松回顧工作流程 (Hackathon Retrospective Workflow)

## 目的

本工作流程旨在引導團隊回顧黑客松過程，總結經驗教訓，規劃未來改進，以促進團隊的持續學習與成長。

## 如何調用

這個工作流程是 `standalone: true`，因此可以直接由使用者調用。

在 BMAD 環境中，您可以透過以下指令啟動它：

```bash
workflow /bmad/mhp/workflows/hackathon-retrospective/
```

## 預期輸入

本工作流程為高互動性，將引導使用者提供輸入。使用者需要準備好：

*   對黑客松過程的客觀觀察和個人感受。
*   願意開放分享和接受回饋的心態。

## 生成輸出

本工作流程將在執行過程中，透過對話引導團隊完成回顧。最終會生成一份摘要報告，記錄回顧的目標、收集的數據、分析的洞察和制定的行動計畫：

*   **預設輸出檔案**：`{output_folder}/hackathon-retrospective-report-{{date}}.md`

## 特殊要求

*   本工作流程為高互動性，需要使用者積極參與對話。
*   建議在黑客松結束後，由整個團隊共同參與此工作流程。
